Webpack:
	- É um module bundler (um empacotador de módulos
	para aplicações JS)
	- Bundler: um arquivo final onde o webpack pega todos os módulos
	e empacota em um único arquivo.
	- Suporta: fontes, CSS, Imagens, HTML, JS, Pulugins e JSON

Configuração do WebPack:
	- Entry Point: Ponto de entrada. Arquivo principal onde ele irá
	mapear todos os módulos
	- Output: É para determinar quais são os bundlesrs que o Webpack
	irá emitir.
	- Loaders: É para permitir que o wepack gerencie arquivos que não são Javascript
	- Plugins: Plugins podem ser utilizados para otimização de pacots, minificação
	injeção de scripts e muito mais
	- Mode: Utilizados para abordagem de configuração zero. É possível
	configurar módulos como production, development ou none.
		Production: trás otimizações internas
		Development: é executado com três plugins: UglifyPlugin
		ModuleConcatenationPlugin e NoEmitOnErrorsPlugin
	
Criação do arquivo webpack.config.js:
	npm i -D webpack webpack-cli

	"build": "webpack --mode production"

	npm i @babel/code babel-loader @babel/preset-env @babel/preset-react --save-dev
